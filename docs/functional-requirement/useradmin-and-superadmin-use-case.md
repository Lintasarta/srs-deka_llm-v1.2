# Useradmin and Superadmin Use Case

# Use Case Description

This is the process both Useradmin and Superadmin interact with the system related to Deka LLM products.

**Action and Response**

| Action by user | Response from system |
|---|---|
| Useradmin can create 1 project on Service Portal and automatically create 1 Team on Lite LLM | System will display project form |
| Useradmin can create 1 or more API key and choose the models on 1 Project | System will display api key creation form and Lite LLM will creates 1 team |
| Default user will automatically add on Lite LLM once the API Key created, and it comes from user creator Service Portal | Lite LLM will creates 1 default user from creator user Service Portal |
| Useradmin can view the list of models that Deka LLM provides | System will display list of models |
| Useradmin can view the list of models that project used | Systems will display the list of models that project used |
| Useradmin can view and delete API Key on the Projects | System will list API KEY and delete the api key |
| Useradmin can view the limit of their Token and Limit request | System will display the limit |
| Useradmin can view the usage on billing module daily cost estimated tab | System will display the usage on daily cost estimated |
| Useradmin can view and download the Invoice on billing module invoice tab | System will display the list of invoice and button download |
| Useradmin can view the report of monthly cost on monthly cost and summary monthly tab | System will display the list of monthly cost and summary monthly |
| Useradmin can view the metrics of usage activity api requests and total tokens | System will display the metrics |
| Superadmin can create 1 project on Service Portal and automatically create 1 Team on Lite LLM | System will display project form pop up |
| Superadmin can create 1 or more API key and choose the models on 1 Project | System will display api key creation form and Lite LLM will creates 1 team |
| Default user will automatically add on Lite LLM once the API Key created, and it comes from user creator Service Portal | Lite LLM will creates 1 default user from creator user Service Portal |
| Superadmin can view the list of models that Deka LLM provides | Systems will display the list of models that project used |
| Superadmin can view the list of models that project used | System will list API KEY and delete the api key |
| Superadmin can view and delete API Key on the Projects | System will display the limit |
| Superadmin can view the limit of their Token and Limit request | System will display the usage on daily cost estimated |
| Superadmin can view the usage on billing module daily cost estimated tab | System will display the list of invoice and button download |
| Superadmin can view and download the Invoice on billing module invoice tab | System will display the list of monthly cost and summary monthly |
| Superadmin can view the metrics of usage activity api requests and total tokens | System will display the metrics |
| Superadmin can create and update the limit on Project | System will display project limit menu |
| Superadmin can view the Audit and Activity Logs | System will display the logs |
